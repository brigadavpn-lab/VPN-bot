## ✅ Шаг 4: Проверка работы

### 1. Проверьте подключение к VPN

**Откройте:** https://whoer.net

**Должно быть:**
- IP адрес: **НЕ ваш реальный IP** (должен быть IP сервера)
- Страна: **Германия или другая** (зависит от локации сервера)
- DNS: **1.1.1.2, 1.0.0.2** или **Cloudflare**
- "Ваша анонимность": высокий уровень

### 2. Проверьте DNS утечки

**Откройте:** https://dnsleaktest.com

**Нажмите:** "Extended test"

**Должно показать:**
- DNS серверы: **Cloudflare (1.1.1.2, 1.0.0.2)**
- Страна DNS: **Не Россия**
- НЕ должны светиться DNS вашего провайдера

### 3. Проверьте Gemini

**Откройте:** https://gemini.google.com

**Результат:**
- ✅ **Должен открыться** интерфейс Gemini
- ✅ Можно задавать вопросы
- ❌ **НЕ должно быть** сообщения "Gemini пока не поддерживается в вашей стране"

### 4. Проверьте split-tunneling (.ru домены)

**Откройте:** https://yandex.ru или любой .ru сайт

**Результат:**
- ✅ Сайт должен открываться **напрямую** (без VPN)
- ✅ Скорость загрузки должна быть нормальной

Для проверки:
- Откройте v2rayN → "Статистика" → смотрите трафик
- Если трафик на .ru сайтах не растет = split-tunneling работает

## 🐛 Если что-то не работает

### Проблема 1: VPN не подключается

**Симптомы:**
- v2rayN показывает ошибки при запуске
- Красный индикатор в трее
- Сайты не открываются

**Решение:**
```
1. Откройте v2rayN
2. Откройте "Параметры" → "Core: Xray / SingBox / ..."
3. Выберите "Xray-core" (если не выбрано)
4. Нажмите "Проверить обновления ядра"
5. Перезапустите v2rayN
```

Если не помогло:
```
1. Закройте v2rayN
2. Удалите config.json
3. Верните config.json.backup.2026-01-28
4. Запустите v2rayN
```

### Проблема 2: VPN работает, но Gemini не открывается

**Решение:**
```
1. Очистите кэш браузера:
   - Chrome/Edge: Ctrl+Shift+Delete
   - Выберите "Весь период"
   - Отметьте "Кэшированные изображения и файлы"
   - Нажмите "Удалить данные"

2. Закройте браузер полностью

3. Откройте браузер заново в режиме инкогнито (Ctrl+Shift+N)

4. Попробуйте открыть https://gemini.google.com
```

Если не помогло:
```
1. Отключитесь от VPN
2. Очистите DNS кэш Windows:
   - Откройте CMD от администратора
   - Выполните: ipconfig /flushdns
3. Подключитесь к VPN заново
4. Попробуйте Gemini снова
```

### Проблема 3: Медленная скорость через VPN

**Решение:**
```
1. В v2rayN откройте "Параметры"
2. "Параметры маршрутизации" → "Режим маршрутизации"
3. Попробуйте разные режимы:
   - "Только для прокси"
   - "Глобальный"
   - "Обходной список"
```

### Проблема 4: .ru сайты идут через VPN (не напрямую)

**Проверка:**
```
1. Откройте v2rayN
2. "Статистика" → смотрите трафик
3. Откройте yandex.ru
4. Трафик в v2rayN не должен расти
```

**Если трафик растет:**
```
Значит .ru домены идут через VPN (неправильно)
Проверьте, что в config.json есть:
{
  "routing": {
    "rules": [
      {
        "type": "field",
        "outboundTag": "direct",
        "domain": ["regexp:.*\\.ru$"]
      }
    ]
  }
}
```

## ✨ Что делает FakeDNS?

**FakeDNS** - это механизм, который:
1. Перехватывает DNS запросы **до** их отправки
2. Выдает временные "поддельные" IP адреса (198.18.x.x)
3. Разрешает реальные домены через **DoH** (шифрованно)
4. Правильно обрабатывает **QUIC, HTTP/3** протоколы

**Почему это важно для Gemini:**
- Gemini использует HTTP/3 через QUIC
- Обычный DNS + sniffing может некорректно обрабатывать QUIC
- FakeDNS решает эту проблему на уровне DNS

**Безопасность:**
- ✅ Все DNS запросы шифруются через DoH
- ✅ Провайдер не видит, какие сайты вы посещаете
- ✅ Защита от DNS spoofing и перехвата
- ✅ Malware домены блокируются (1.1.1.2, 1.0.0.2)
