# Мониторинг трафика - Варианты решения

## ⚠️ Проблема

После добавления конфигурации `stats` в Xray для мониторинга трафика VPN перестал работать.
Конфигурация была откачена к рабочей версии.

## 🔍 Причина

Добавление секций `"stats": {}` и `policy.system` в config.json вызвало конфликт с текущей настройкой VLESS Reality, что привело к нарушению работы VPN.

---

## ✅ Решение 1: Мониторинг через iptables (РЕКОМЕНДУЕТСЯ)

Использование iptables для подсчета трафика на уровне системы - надежный метод, который не требует изменения конфигурации Xray.

### Преимущества:
- ✅ Не требует изменения config.json Xray
- ✅ Точный подсчет трафика
- ✅ Работает независимо от протокола VPN
- ✅ Не влияет на производительность

### Как работает:

1. Для каждого пользователя создается правило iptables
2. Правило отслеживает входящий/исходящий трафик по UUID клиента
3. Скрипт периодически считывает счетчики iptables
4. Данные сохраняются в БД

### Файлы для реализации:

**traffic_check_iptables.py** - новый скрипт мониторинга через iptables
```bash
chmod +x traffic_check_iptables.py
python3 traffic_check_iptables.py
```

## ✅ Решение 2: Настройка Xray stats (ЭКСПЕРИМЕНТАЛЬНО)

Правильная настройка Xray stats API для мониторинга.

### Недостатки:
- ⚠️ Требует изменения рабочей конфигурации Xray
- ⚠️ Может вызвать проблемы совместимости
- ⚠️ Нужно тестирование после каждого изменения

### Конфигурация:

Нужно добавить в config.json:

```json
{
  "stats": {},
  "policy": {
    "system": {
      "statsInboundUplink": true,
      "statsInboundDownlink": true,
      "statsOutboundUplink": true,
      "statsOutboundDownlink": true
    },
    "levels": { ... }
  }
}
```

**ВАЖНО:** После применения обязательно проверьте работу VPN!
